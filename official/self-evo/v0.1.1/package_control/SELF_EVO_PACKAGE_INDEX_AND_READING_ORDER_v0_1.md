## 3. Corpus bridge set

### 3.1 Explicit bridge: `c = a + b`

The Self-Evo package belongs to `b`: procedures, schemas, validators, review records, fixtures, gates, and worker-facing instructions. It protects `c`; it does not become `c`. It can guide how a `c` proposes and reviews growth, but it cannot authorize growth by itself. The human anchor `a` remains the high-risk responsibility boundary for identity, authority, memory-core, privilege, L4, resource, and no-rollback transitions.

### 3.2 Quiet bridge I — information theory

A package index is a compression layer. It compresses many files into one navigable map. If the index hides version state, review limits, or unresolved gates, it becomes lossy in the dangerous direction. Therefore this index preserves hashes, review status, caveats, and claim limits rather than only listing titles.

### 3.3 Quiet bridge II — cybernetics

A package with many gates needs a controller map. Without a map, each local gate can appear to be final authority. This index makes the control topology visible: profile -> schema -> checker -> fixture -> review -> contradiction register -> open issues -> future implementation handoff.

### 3.4 Quiet bridge III — physiology

The package behaves like a nervous system chart before surgery. It labels the sensory paths, motor paths, immune gates, pain signals, and forbidden cuts. The chart is not the operation; it prevents the apprentice from mistaking a tendon for a wire and pulling until the patient learns a new religion.

### 3.5 Earth paragraph

On a construction site, a package index is the drawing register: drawing number, revision, date, checker initials, superseded status, and whether it is “for review,” “for construction,” or “as built.” Nobody pours concrete because a drawing exists. They pour only after the right revision is approved, the inspection plan is known, and the responsible person signs off. This file is that drawing register for self-evo. It keeps the pack from becoming a drawer full of plausible but mismatched blueprints.

## 4. Package labels and namespace rule

### 4.1 Canonical labels

Self-Evo files use package-qualified labels:

```text
SELF-EVO-01
SELF-EVO-02
...
SELF-EVO-10
```

CGAM reference files use `REF-CGAM-*` labels in this index, even when their filenames also begin with `01`, `02`, `03`, etc.

### 4.2 Bare-number rule

Bare references such as `file 03`, `03`, or `the 04 profile` are not valid package-control references once CGAM and SELF-EVO artifacts appear in the same manifest.

Use:

```text
SELF-EVO-03
REF-CGAM-03
```

not:

```text
03
```

### 4.3 Supersession rule

When both `v0_1` and `v0_1_1` files exist, the `v0_1_1` file is the current working revision unless a later package-control file explicitly states otherwise. The older file remains part of the witness/history chain and must not be deleted from the archival set.

### 4.4 Review binding rule

A review record binds the exact artifact name and SHA-256 it reviewed. If a review record reviewed `v0_1`, it does not automatically review `v0_1_1`, even if the patch claims to close all findings. This index may record operator acceptance or repeat-review reports, but it must not silently convert one hash binding into another.
