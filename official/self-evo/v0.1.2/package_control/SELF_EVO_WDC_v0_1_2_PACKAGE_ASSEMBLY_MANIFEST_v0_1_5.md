## 1. Corpus bridge set

### 1.1 Explicit bridge: `c = a + b`

The assembled WDC package belongs to `b`: documents, manifests, review records, gate records, patch notes, checksums, and package-control routes. It serves `c` by preserving the conditions under which witness-dependency capture is visible and challengeable.

The package does not become `c`. It does not speak as `a`. It does not authorize a system to evolve itself.

### 1.2 Quiet bridge I — information theory

A final package is a high-compression object. The danger is that the compression hides unresolved implementation work or turns evidence into certification. This manifest therefore carries hashes, gate states, nonclaims, and retained open issues explicitly so the package remains information-preserving.

### 1.3 Quiet bridge II — cybernetics

The assembly manifest is the package's control-panel state. If a gate closes but the panel still shows pending, or if a gate remains open but the panel shows green, the control loop lies. This manifest synchronizes the WDC gate state with the assembled package.

### 1.4 Quiet bridge III — physiology

A healed wound and a discharged patient are not the same thing. The WDC package has repaired its document-level seams and closed its package-control checks, but implementation and runtime claims remain contraindicated. The assembly is a clean chart, not permission to run.

### 1.5 Earth paragraph

A handover binder can be complete while the building is not opened to the public. It may contain drawings, inspection records, redlines, punch lists, and sign-offs, yet still say: not commissioned, not occupied, not energized. This manifest is that binder for WDC v0.1.2.
