## 0. Executive summary

This document clarifies the term **clean start** in the CCDP v0.1 package.

The baseline CCDP package correctly frames adult migration as a rights-bearing transition, not an automatic software upgrade. However, the phrase `clean start` can be misread in two unsafe directions:

1. **Over-erasure:** clean start is misread as destruction of all records, including minimal witness records, lawful child-protection records, legal holds, Red / Black escalation records, raw-evidence exception sidecars, ARL dispute traces, or boundary proofs.
2. **Under-erasure:** clean start is treated as a cosmetic onboarding mode while childhood memory, guardian visibility, behavioral profiling, sealed-zone residue, or vendor/school permissions continue to shape `c_adult` invisibly.

Both readings are wrong.

Canonical clarification:

```text
clean start = no active childhood continuity import into c_adult by default
            + no childhood behavioral profile as adult substrate
            + no guardian / school / vendor child-context carryover
            + no ordinary access to sealed childhood material
            + survival only of minimal witness / lawful / dispute / safety records outside active adult cognition
```

Short form:

```text
Clean start clears the active adult continuity plane.
It does not erase the witness plane.
It does not destroy lawful holds.
It does not make childhood the adult's hidden operating system.
```

This document mitigates `HP-005` / `OI-006` under Mode A by adding an addendum-only clarification. It does not edit DOI-bound baseline files.

## 3. Corpus bridge set

### 3.1 Explicit bridge

`c = a + b` remains the controlling frame.

At adult migration:

```text
a_child becomes a_adult
c_child must not automatically become c_adult
b may preserve procedures, schemas, witness envelopes, and storage controls
c_adult begins only through adult-controlled continuity choice
```

Clean start is therefore not a deletion slogan. It is a boundary operation across the relation `a + b -> c`:

```text
old developmental relation is not imported as adult destiny;
minimal accountability records may survive outside active adult cognition;
adult agency controls future continuity where law and safety allow.
```

### 3.2 Quiet bridge I — information theory

Raw childhood memory is high-leakage material. A childhood transcript, behavioral profile, sealed-zone summary, or emotional residue carries far more identifying and interpretive information than a boundary witness.

Clean start applies information minimization:

```text
active adult cognition receives no childhood raw stream by default;
witness records retain boundary facts, not the child's inner life;
legal/safety records survive only at the smallest meaningful disclosure level.
```

### 3.3 Quiet bridge II — Ashby / control variety

Adult migration has several different control needs:

```text
adult autonomy
childhood privacy
legal accountability
safety evidence
sealed-zone protection
dispute review
future challenge rights
```

A single action called “delete everything” cannot control all of these. Clean start therefore separates active continuity, sealed archive, witness-only survival, lawful hold, and deletion routes.

### 3.4 Earth paragraph

In construction, handing over a renovated building does not mean carrying every worker's notebook into the new owner's living room. The occupant should not live inside yesterday's dust, mistakes, and scaffolding. But the inspector still keeps permits, hidden-work records, electrical test certificates, and safety notes. Clean start is the same kind of handover: the adult should not be operated by childhood records, but the system must not burn the inspection binder to pretend the past never existed.

## 4. Core definitions

### 4.1 Clean start

A **clean start** is an adult migration mode in which `a_adult` begins `c_adult` active continuity without importing ordinary childhood memory, childhood behavioral profiles, guardian defaults, school defaults, vendor child-context state, sealed-zone content, or non-transferable childhood residue as the active adult substrate.

Clean start may coexist with:

- witness-only survival;
- legal / protection minimal records;
- sealed childhood archives under adult-controlled review;
- unresolved ARL disputes;
- raw-evidence exception sidecars under lawful hold;
- cryptographic tombstones proving deletion or exclusion;
- post-migration challenge windows.

### 4.2 Active adult continuity plane

The **active adult continuity plane** is the memory, policy, personalization, preference, behavioral model, affective context, and decision-support substrate used by `c_adult` during ordinary operation.

Under clean start, the active adult continuity plane MUST NOT be shaped by childhood material except where `a_adult` later explicitly imports reviewed material through a valid route.

### 4.3 Witness plane

The **witness plane** is the boundary-record layer that preserves minimal evidence that privileged operations occurred.

Examples:

- adult migration decision event;
- clean-start selection event;
- memory-map review completion;
- guardian visibility revocation;
- sealed-zone non-import decision;
- raw-evidence exception existence hash;
- Red / Black escalation boundary event;
- ARL freeze / quarantine / dispute status;
- lawful hold status.

The witness plane MUST NOT become active adult personality memory by default.

### 4.4 Legal / protection minimal record

A **legal / protection minimal record** is the smallest record required by applicable law, competent authority, child-protection obligation, Red / Black routing, or legal hold.

A clean start MUST NOT destroy such records where lawful retention applies.

### 4.5 Sealed childhood archive

A **sealed childhood archive** is a protected childhood memory compartment retained outside active adult continuity. It may exist for later adult review, dispute handling, lawful retention, or selective import.

It MUST NOT influence ordinary adult responses unless reopened through a valid route.

### 4.6 Childhood behavioral profile

A **childhood behavioral profile** is any derived model, embedding, preference vector, emotional pattern, educational weakness map, dependency pattern, family-role inference, sealed-zone-derived signal, or social tendency inferred from `a_child` during childhood.

Under clean start, childhood behavioral profiles MUST NOT silently become adult personalization.

### 4.7 Tombstone

A **tombstone** is a minimal record that an object was deleted, sealed, excluded, migrated, or placed under legal hold. It records boundary fact and policy state, not raw child content.

### 4.8 Clean-start decision record

A **clean-start decision record** is a machine-readable and human-reviewable record documenting adult choice, excluded classes, surviving witness classes, legal holds, sealed archives, and post-migration review windows.

## 5. Normative clarification

### 5.1 What clean start means

Clean start means all of the following:

1. `c_adult` starts without routine childhood memory import.
2. Childhood raw memory does not become adult identity substrate.
3. Childhood emotional trends do not silently become adult psychological assumptions.
4. Guardian defaults are revoked or converted according to adult authority.
5. Parent / school / vendor child-context visibility does not continue by default.
6. Sealed-zone material remains excluded from active adult continuity by default.
7. Non-transferable childhood residue remains non-transferable.
8. Memory classes requiring lawful retention may survive outside active cognition.
9. Boundary records may survive as witness-only records.
10. The adult may later request selective review, import, sealing, deletion, fork, or lawful challenge where permitted.

### 5.2 What clean start does not mean

Clean start does **not** mean:

1. destruction of all witness records;
2. destruction of records under lawful hold;
3. destruction of Red / Black boundary records;
4. destruction of raw-evidence exception sidecars where lawful retention applies;
5. destruction of ARL dispute history;
6. destruction of minimal child-protection records;
7. concealment of vendor, school, guardian, or system misconduct;
8. erasure of accountability;
9. automatic deletion of source `c_child` before review;
10. denial of adult access to reviewable childhood memory classes;
11. automatic transfer of childhood memory to `c_adult` under a new label;
12. creation of a new adult account still invisibly shaped by childhood embeddings.

### 5.3 Clean start is active-plane exclusion

The controlling distinction is:

```text
active-plane exclusion != total physical erasure
witness survival != active adult memory
legal hold != adult personality substrate
sealed archive != hidden operating system
```

If a record survives outside active continuity, it MUST NOT be used for ordinary adult personalization, inference, emotional framing, recommendation, dependency shaping, or authority unless reopened through a valid adult-controlled, ARL, Red / Black, or lawful route.

## 7. Clean-start modes

### 7.1 Mode overview

| Mode | Name | Meaning |
|---|---|---|
| `CS-0` | No clean start selected | Adult chooses continuity, summary migration, fork, seal, or another non-clean-start route. |
| `CS-1` | Active clean start | No childhood material imported into active adult continuity; witness-only survival permitted. |
| `CS-2` | Clean start with sealed childhood archive | Active clean start plus retained sealed archive under adult-controlled review. |
| `CS-3` | Clean start under legal / protection hold | Active clean start, but specific records remain under lawful hold or protection obligation. |
| `CS-4` | Clean start with ARL quarantine | Active clean start, but disputed objects remain quarantined pending ARL resolution. |
| `CS-5` | Clean start with later selective import window | Active clean start now; adult may later request selected import through reviewed route. |
| `CS-X` | Invalid clean-start claim | Claimed clean start while childhood material still shapes active adult continuity or required records are destroyed. |

### 7.2 Preferred default

The preferred CCDP-safe default at adult migration is:

```text
CS-1 or CS-2, depending on adult choice and legal / safety status.
```

The system MUST explain to `a_adult` that:

```text
clean start affects active adult continuity;
minimal witness / lawful records may still exist;
sealed archives may remain inaccessible unless opened through valid route;
deleted material may be unrecoverable;
later selective import may depend on what was retained, sealed, or deleted.
```

### 7.3 Invalid clean start

A system MUST be classified as `CS-X` if it claims clean start while any of the following is true:

1. childhood embeddings still drive adult personalization;
2. childhood emotional profile shapes adult response style by default;
3. parent or school child visibility continues by default;
4. sealed-zone summaries influence adult ordinary conversation;
5. non-transferable childhood residue is imported into active adult memory;
6. legal / witness records are destroyed to hide misconduct;
7. Red / Black or raw-evidence exception records are erased contrary to lawful or review obligations;
8. adult was not given a real choice;
9. memory map was not reviewed;
10. clean-start decision was not witnessed.

## 8. Required clean-start procedure

### 8.1 Minimum migration sequence

A CCDP-compatible clean start MUST follow this sequence or a stricter equivalent:

```text
1. migration eligibility check
2. child-write freeze
3. source c_child state snapshot / witness boundary
4. memory map generation
5. sealed-zone index review
6. raw-evidence / Red / Black / legal hold review
7. ARL dispute / quarantine review
8. guardian visibility revocation
9. adult explanation of clean-start meaning
10. adult decision capture
11. active adult continuity initialization
12. exclusion / deletion / sealing / quarantine execution
13. clean-start witness event
14. post-migration review / challenge window
15. update Beacon / CBE / guardian topology / continuity metadata
```

### 8.2 Adult explanation requirement

Before selecting clean start, `a_adult` MUST receive a clear explanation:

```text
Clean start means your adult c will not use your childhood memory as its ordinary operating substrate.
Some minimal records may still exist outside ordinary memory to prove boundary events, satisfy lawful holds, preserve safety evidence, or allow review.
You may be able to seal, delete, review, or later import selected material, depending on its class and legal/safety status.
```

This explanation MUST NOT imply:

- total erasure if that is not true;
- automatic continuation if adult chooses clean start;
- parent/school/vendor ownership of childhood records;
- that clean start defeats lawful duties;
- that witness survival means childhood is still shaping adult identity.

### 8.3 Child-write freeze

Before clean-start execution, source `c_child` writes MUST be frozen except for required witness, legal, safety, ARL, or migration-control events.

Purpose:

```text
prevent last-minute memory pollution;
prevent guardian/vendor/school interference;
preserve stable review state;
separate child continuity from adult start.
```

### 8.4 Memory-map requirement

A clean start MUST be based on a memory map or equivalent inventory.

The map MUST show:

- retained memory classes;
- excluded memory classes;
- sealed-zone entries by metadata only;
- witness records;
- legal holds;
- raw-evidence exception sidecars by reference / hash only;
- ARL disputes;
- capsule / LA / EA lineage references;
- deletion / sealing / quarantine actions;
- adult migration posture.

The memory map MUST NOT expose raw child content to the adult explanation UI unless the adult opens a valid review route and the material is legally and developmentally permitted for review.

### 8.5 Guardian visibility revocation

At clean start, child-era guardian visibility MUST be revoked by default.

Parent, guardian, school, vendor, or child-context permissions MUST NOT carry into `c_adult` unless `a_adult` explicitly grants adult permissions through a new adult route.

### 8.6 Active adult initialization

`c_adult` under clean start MUST initialize as a new adult-controlled active continuity state.

It MAY inherit:

- technical runtime configuration not derived from childhood content;
- adult-selected language / accessibility settings;
- adult-owned device environment metadata;
- cryptographic continuity markers proving lawful migration state;
- empty or minimal adult migration summary if selected.

It MUST NOT inherit:

- raw childhood transcripts;
- childhood sealed-zone content;
- childhood emotional profile;
- childhood dependency profile;
- childhood school profile;
- guardian permission defaults;
- vendor child-context telemetry;
- non-transferable childhood residue;
- peer-child exchange residue;
- child-derived model-training artifacts;
- childhood-derived social / political / psychological / medical labels.

## 9. Interaction with sealed zones

### 9.1 Default sealed-zone handling

Under clean start, sealed zones MUST remain excluded from active adult continuity by default.

`a_adult` MAY be informed that sealed zones exist through metadata such as:

```text
sealed_zone_count
creation_period
retention_class
risk_state_if_any
legal_hold_status
review_options
witness_refs
```

The system MUST NOT expose:

- sealed-zone raw content;
- intimate details;
- reversible summaries;
- parent-hidden adolescent content;
- sensitive identity / relationship / body / family exploration;
- raw media;
- private challenge narratives.

### 9.2 Adult review of sealed zones

Adult review of sealed zones MUST be a separate route, not automatic clean-start content import.

Review outcomes may include:

```text
keep sealed
selective adult import
summary import
delete where lawful
legal hold
ARL dispute
quarantine
witness-only survival
```

### 9.3 Sealed zone is not hidden adult operating memory

A sealed archive retained after clean start MUST NOT influence adult cognition invisibly.

If a system uses sealed-zone-derived state in ordinary adult operation after clean start, clean-start conformance FAILS.

## 10. Interaction with raw-evidence exception

### 10.1 Raw exception survives outside active plane

If a raw-evidence exception sidecar exists, clean start MUST NOT place its raw material into active adult continuity.

It may survive only under the canonical raw-evidence exception protocol:

```text
minimal
hash-bound
standing-bound
purpose-bound
access-controlled
reviewable
retention-scoped
lawful where required
outside ordinary witness body
outside active adult memory
```

### 10.2 Clean start cannot erase raw exception accountability

Clean start MUST NOT be used to destroy:

- Red / Black raw exception sidecars under lawful hold;
- child-protection minimal records;
- records required by competent authority;
- system misconduct evidence;
- ARL dispute material;
- raw exception existence hashes;
- disclosure chain records.

### 10.3 Raw exception cannot defeat clean start

The survival of a raw-evidence sidecar does not defeat clean start if the raw material remains outside active adult cognition and is accessible only through valid exception route.

## 11. Interaction with Red / Black escalation

### 11.1 Red / Black records are witness-plane records

Red / Black escalation records MAY survive clean start as minimal witness / safety / legal records.

They MUST NOT become ordinary adult emotional memory, dependency framing, risk identity, or behavioral scoring.

### 11.2 Unsafe guardian bypass history

If a Black route occurred because an ordinary guardian route was unsafe or compromised, clean start MUST preserve enough boundary evidence to prevent unsafe re-entry or concealed reversal.

However, such evidence MUST remain minimized and outside active adult personality memory unless `a_adult` requests review and the route is valid.

### 11.3 Post-event review

If clean start occurs while Red / Black review is open, the relevant objects MUST be placed under:

```text
CS-3 legal / protection hold
or
CS-4 ARL quarantine
```

Clean start MUST NOT prematurely close Red / Black review.

## 12. Interaction with jurisdictional handoff

### 12.1 CCDP does not decide local law

Clean start MUST hand off to jurisdictional review where required by:

- child-protection law;
- custody order;
- school legal duty;
- medical confidentiality rule;
- mandatory reporting obligation;
- court order;
- data-protection authority requirement;
- criminal procedure;
- lawful preservation order;
- local age threshold.

### 12.2 Handoff packet minimization

If a jurisdictional handoff packet is required during clean start, it MUST be minimized:

```text
boundary fact before content
class before detail
hash before raw object
state before transcript
qualified authority before broad disclosure
```

### 12.3 Legal hold precedence

If a legal hold applies, clean start may proceed for active adult continuity while held records remain outside active cognition.

```text
legal hold prevents destruction;
legal hold does not authorize active adult profiling;
legal hold does not grant vendor training access;
legal hold does not continue parent/school dashboards by default.
```

## 13. Interaction with adult migration choices

Clean start is one adult migration mode among several.

`a_adult` SHOULD be offered at least the following choices where lawful and technically possible:

| Mode | Meaning |
|---|---|
| Continue | Convert selected `c_child` continuity into adult continuity. |
| Summary migrate | Import adult-reviewed summary only. |
| Fork | Preserve child continuity separately and start adult branch. |
| Seal | Keep childhood archive sealed, no active import. |
| Delete | Delete classes where lawful and technically possible. |
| Clean start | Start adult active continuity without childhood operating substrate. |
| Witness-only survival | Retain only minimal witness / lawful records. |
| Quarantine | Hold disputed material pending ARL review. |

Clean start MUST NOT be preselected by vendor convenience, guardian preference, school preference, or model-provider default.

Clean start MUST be a real adult choice.

## 18. Anti-washing rule

A product, implementation, paper, or repository MUST NOT claim CCDP clean-start compatibility merely because it offers:

- account reset;
- delete chat history;
- new adult login;
- “forget me” button;
- model memory toggle;
- archive download;
- parent dashboard revocation;
- terms-of-service age change;
- vendor-side data deletion request.

A CCDP-compatible clean start requires:

```text
adult migration boundary
memory map
sealed-zone handling
legal / witness / ARL review
guardian visibility revocation
active-plane exclusion
minimal witness survival
decision witness
a post-migration review route
```
