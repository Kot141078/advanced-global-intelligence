# CCDP GLOSSARY

**Package:** Child-`c` Development Protocol
**Version:** v0.1
**Date:** 2026-05-14

---

## 0. Purpose

This glossary defines the core terms used across the CCDP package.

It does not replace the parent corpus glossary. It clarifies child-specific usage and disambiguates overloaded terms such as `C0`, `CCDP-0`, and `C-A4`.

---

## 1. Core terms

| Term | Definition |
| --- | --- |
| `a` | The accountable human anchor in `c = a + b`. In CCDP, this may be `a_child` or `a_adult` at migration. |
| `a_child` | Developing human subject: not property, not a data source, and not a fully autonomous adult actor. |
| `a_adult` | The same human after adult migration threshold under applicable jurisdiction. |
| `b` | The technological substrate: procedures, models, memory, compute, interfaces, and infrastructure. |
| `c` | Persistent AI entity or presence emerging from `a + b` under constraints and continuity. |
| `c_child` | Child-bound persistent AI presence under CCDP constraints. |
| `c_adult` | Adult-controlled continuity after migration, fork, clean start, or reviewed continuation. |
| `c_parent` | Parent or guardian-side AI presence. Guardian support, not owner of child inner life. |
| `c_school` | School-facing AI system or institutional educational entity. |
| CCDP | Child-c Development Protocol; child-specific profile over the existing `c = a + b / SER / L4` corpus. |
| L4 | Reality Boundary Layer: cost, time, energy, scarcity, identity, irreversibility, physical and operational constraints. |
| SER | Sovereign Entity Recursion; persistent entity architecture under real constraints. |
| Beacon | Inter-entity recognition profile; recognizable continuity signal, not a child registry. |
| CBE | Child Beacon Extension; child-specific permission overlay over Beacon recognition. |
| AGL | Actor Grounding Layer; source-state and actor grounding before reliance. |
| ARL | Arbitration / Review Layer; standing, admissibility, freeze, quarantine, review, and re-entry. |
| `c[q]` | Non-collapse state under uncertainty: signal is not automatically memory, evidence, diagnosis, command, or outcome. |
| Soft Safety | Privacy-preserving safety signaling: state, not content; smoke detector, not camera. |
| State signal | Structured support cue emitted without transcript disclosure by default. |
| Red route | Urgent risk route through a safe guardian or emergency path. |
| Black route | Urgent risk route that bypasses ordinary guardian when that route may be unsafe or compromised. |
| Sealed zone | Protected adolescent memory compartment hidden from ordinary parent, school, vendor, and external access. |
| Adult migration | Reviewable transition from `c_child` to adult-controlled continuity. |
| Clean start | Adult choice to start active adult continuity without raw childhood archive as operating substrate; not unlawful erasure of minimal witness/legal records. |
| Memory Map | Class-level machine-readable inventory of child memory, visibility, retention, sealed zones, witness links, and migration posture. |
| Witness event | Tamper-evident boundary record proving privileged transitions without logging the child’s raw inner life by default. |
| VXCX | Visual eXperience Capsule eXchange; structured experience exchange without raw pixels by default. |
| LA | Learning Abstract; capability-improving abstraction that does not carry legitimacy or authority. |
| EA | Experience Artifact; consequence-bearing, L4-confirmed, witness-backed artifact that may carry bounded authority. |
| Raw child life | Transcripts, emotional diary, family conflict, sealed-zone content, face/voice/sensor streams, or intimate childhood material. |
| External agent | Any non-local or non-primary system attempting child-facing interaction or influence. |
| Peer-`c` | Another child’s `c_child` in peer play, learning, or shared contexts. |
| Physical agent | Toy, robot, NPC body, room device, sensor, actuator, or embodied child-facing system. |
| Oracle addiction | Instant ask-answer-relief loop that reduces uncertainty tolerance and agency. |
| Dependency audit | Privacy-preserving aggregate review of child-system dependency risk. |
| Guardian | Actor with bounded responsibility or standing around the child. Guardian does not mean owner. |
| Jurisdictional handoff | Point where CCDP must stop making architectural assumptions and route to local law or competent authority. |
| CCDP-washing | Claiming CCDP compatibility without conformance evidence and witnessable boundaries. |

## 2. Classification disambiguation

| Name | Meaning | Example |
|---|---|---|
| Child maturity level | Developmental / operational maturity level of `c_child` | `C0`, `C1`, `C2`, `C3`, `C4`, `C5` |
| CCDP conformance class | Product/system conformance claim | `CCDP-0`, `CCDP-1`, `CCDP-5`, `CCDP-X` |
| Assertion class | Corpus-strength / evidence class | `C-A4`, `C-A7`, `C-A10` |
| Risk state | Soft Safety / escalation state | `Green`, `Yellow`, `Orange`, `Red`, `Black`, `Unknown` |
| Dependency level | Dependency / oracle-addiction risk | `D0`, `D1`, `D2`, `D3`, `D4`, `DU` |
| Memory class | Child memory class | `M0`…`M12` |

## 3. Preferred wording

Use:

```text
state signal
child-facing persistent AI presence
child-specific overlay
adult migration
sealed zone
minimal disclosure
witness the boundary, not the child
```

Avoid:

```text
parental control AI
child surveillance AI
AI babysitter
robot friend as default
therapy bot
child profile database
permanent childhood archive
```
