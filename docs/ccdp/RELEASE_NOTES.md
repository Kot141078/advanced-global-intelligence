## 0. Release summary

This release introduces **CCDP — Child-`c` Development Protocol**, a child-facing persistent AI entity protocol pack aligned with the existing `c = a + b / SER / L4` corpus.

CCDP defines how a child-bound AI presence may be raised, constrained, recognized, disputed, witnessed, memory-limited, and later migrated into adult-controlled continuity.

The package is designed for review by architects, safety researchers, child-development specialists, educators, legal/compliance reviewers, and implementation engineers.

---

## 1. Main contribution

The release establishes a protocol frame for the coming environment where children may encounter:

- personal `c_child` presences;
- generated media and adaptive cartoons;
- toys and robots with AI personalities;
- school agents and learning platforms;
- peer-`c` systems;
- external agents and synthetic companions;
- persistent memory systems.

The central claim is:

> A child-facing `c` must be close enough to support development and bounded enough that the child can leave it, challenge it, outgrow it, or continue it as a choice.

---

## 2. Load-bearing principles

1. CCDP is an overlay, not a new root stack.
2. Beacon recognizes continuity; CBE restricts child-facing permission.
3. AGL grounds source and actor before reliance.
4. ARL handles disputes, standing, freeze, quarantine, review, and re-entry.
5. Soft Safety signals state, not content.
6. Witness records prove boundaries, not the child's inner life.
7. Raw child life is not exchange material.
8. Memory is developmental, not archival.
9. Adult migration is a rights-bearing transition, not automatic software upgrade.
10. Red protects through a safe route; Black bypasses unsafe routes.
