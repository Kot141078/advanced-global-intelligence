**Location:** Brussels

## Abstract

Not every Experience Artifact is economically admissible, transferable, or reusable.

Value requires:
- provenance,
- standing,
- bounded disclosure,
- dispute-aware exchange semantics,
- source-class discipline,
- and fail-closed enforcement.

This document therefore defines the first normative economic layer for Experience Artifacts (EA).

It does not create a market of experience.
It defines:
- admissibility into circulation,
- value classes,
- transfer modes,
- provenance requirements,
- disclosure and privacy boundaries,
- escrow / circulation hold semantics,
- ARL hooks,
- anti-gaming constraints,
- and fail-closed restraint.

---

## 1. Purpose

The purpose of this layer is to define how Experience Artifacts may become economically legible
without collapsing into:
- premature pricing,
- unrestricted trade,
- synthetic laundering,
- continuity appropriation,
- or bazaar logic.

The layer exists to answer a practical question:

> Under what conditions may an Experience Artifact matter economically,
> move in bounded form,
> remain restrained,
> or be denied circulation without being treated as worthless?

---

## 2. Scope

This document covers, at minimum:

- EA circulation admissibility
- value classes
- provenance and source-origin requirements
- transfer modes
- receiver-mode and fidelity discipline
- disclosure and privacy boundaries
- escrow / circulation hold semantics
- ARL dispute hooks
- reuse, derivation, contamination, and synthetic laundering risk
- non-transferability pressure
- lineage-linked value and risk
- non-numeric and non-market value handling

---

## 3. Non-goals

v0.1 does **not**:

- create an open exchange
- define universal market price
- require tokenization
- require ranking or scalar commensuration
- define final civil ownership law
- define inheritance markets
- define full bond economics
- replace ARL
- replace witness discipline
- or imply that all EA should become transferable

This layer is not a bazaar design.
It is a bounded circulation RFC.

## 4. Definitions

### 4.1 Experience Artifact (EA)
A structured artifact carrying experience-bearing significance under the wider stack.

### 4.2 Economically admissible EA
An EA that may lawfully participate in one or more economic-layer functions such as:
- bounded allocation,
- escrow,
- preservation,
- transfer under scope,
- limited reuse,
- or circulation priority.

### 4.3 Non-admissible EA
An EA that lacks sufficient basis for the requested economic-layer action.

### 4.4 Value class
A bounded classification of how and why an EA matters economically,
without requiring a universal price.

### 4.5 Provenance chain
The declared and reviewable chain linking:
- origin,
- witness path,
- source class,
- disclosure class,
- scope,
- and restrictions.

### 4.6 Transfer
A bounded movement of custody, presentation standing, or admitted circulation standing
under explicit mode constraints.

### 4.7 Derivation
Creation of a downstream artifact from an upstream EA,
with explicit provenance consequences and no silent parity assumptions.

### 4.8 Escrow
A bounded holding state in which the EA is preserved
but withheld from stronger economic circulation paths pending explicit conditions.

### 4.9 Circulation hold
A narrower restraint blocking one or more circulation paths
without necessarily requiring full escrow or full dispute.

### 4.10 Contested EA
An EA whose circulation, meaning, release, or standing-sensitive use
has become procedurally contested.

### 4.11 Non-transferable EA
An EA whose movement would create unacceptable risk of:
- continuity appropriation,
- standing laundering,
- lineage laundering,
- unsafe disclosure,
- or scar appropriation.

### 4.12 Contaminated EA
An EA whose source class, provenance, or derivation state
makes its current authority posture unsafe or presumptively degraded.

### 4.13 Bounded disclosure
A release of only the necessary representation of the EA
under explicit disclosure and receiver-mode constraints.

### 4.14 Receiver standing
The lawful basis for a receiver to enter a reception path.

### 4.15 Receiver mode
The declared mode under which a receiver handles an EA:
- custody,
- archival retention,
- escrow,
- inspection,
- relay,
- bounded review,
- licensed reuse,
- or derivative transformation.

## 5. Admissibility into circulation

### 5.1 Core rule
An EA may enter economic circulation only if the requested circulation path is lawfully admissible.

### 5.2 Minimum admissibility questions
At minimum, the system must ask:

1. What is the current admission state?
2. What is the source-origin class?
3. What witness / provenance basis exists?
4. What transfer or receiver mode is requested?
5. What non-transferability pressure exists?
6. Is current movement lawful under scope and jurisdiction?
7. Is the artifact under restraint, dispute, freeze, or unresolved maturation?

### 5.3 No automatic circulation rule
No EA enters circulation by default merely because it:
- exists,
- is stored,
- is well formatted,
- or seems useful.

### 5.4 Economic admissibility is narrower than value
An EA may be economically meaningful while still failing the requested circulation path.

---

## 6. Value classes

### 6.1 General rule
The layer should prefer **value classes before prices**.

### 6.2 Illustrative value families
EA may carry:
- contextual value
- relational value
- scar-bearing value
- survival value
- institutional value
- memory value
- precedent value
- lineage-amplified witness value
- relay or archival value
- non-numeric and incommensurable value

### 6.3 Value is not forced into one metric
Value may remain:
- non-numeric,
- domain-bound,
- receiver-bound,
- lineage-bound,
- or incommensurable.

### 6.4 Economic admissibility does not imply marketability
A value-bearing EA may still remain:
- escrowed,
- inspection-only,
- non-transferable,
- or permanently non-market.

---

## 7. Transfer modes

### 7.1 Core rule
Transfer mode semantics are subordinate to admission state, provenance, and jurisdictional activation.

### 7.2 Canonical transfer modes
This layer assumes the following minimal mode vocabulary:

- `assignment_transfer`
- `partial_transfer`
- `bounded_disclosure_transfer`
- `licensed_reuse`
- `inspection_only_non_transferable`
- `review_submission_escrow`

### 7.3 Default baseline
Before activation, the default state is:
- `local_custody_only`

### 7.4 No silent upgrade rule
No transfer mode may silently upgrade:
- admissibility,
- standing,
- scar-bearing force,
- or authority.

### 7.5 Movement is not equivalence
Movement of the artifact does not mean movement of:
- truth,
- scar,
- lineage rights,
- or admitted force.

Binding effect must be re-recognized, or it is not transferred at all.

---

## 8. Provenance requirements

### 8.1 Core rule
No provenance -> no serious circulation claim.

### 8.2 Minimum provenance chain
A minimum circulation-grade provenance chain should preserve:
- creator / originating path
- source-origin class
- witness references where applicable
- continuity link
- disclosure classification
- dispute / restraint status
- transfer restrictions
- integrity proof or linked integrity evidence

### 8.3 Source-origin classes matter
At minimum, the system must distinguish:
- `ORIGIN_LIVED_PRIMARY`
- `ORIGIN_LIVED_DERIVED`
- `ORIGIN_MIXED_SYNTHETIC`
- `ORIGIN_SYNTHETIC_DERIVED`
- `ORIGIN_SYNTHETIC_RESIDUE`

### 8.4 Provenance required, parity not guaranteed
Even provenanced synthetic-derived artifacts do not receive default parity with lived primary origin.

### 8.5 Invalid circulation cases
The following should fail closed or downgrade:
- orphaned EA
- provenance gap
- unsigned or broken witness chain
- synthetic residue presented as clean
- privilege-breached extraction
- disputed artifact presented as release-ready
- lineage-rich artifact presented as lineage-neutral
- high-density artifact transferred without declared receiver mode

---

## 9. Disclosure and privacy boundaries

### 9.1 Value does not imply full disclosure
An EA may be valuable without being fully revealable.

### 9.2 Bounded disclosure rule
The layer should prefer:
- bounded disclosure,
- redaction,
- selective reveal,
- and explicit receiver-mode typing.

### 9.3 Disclosure and transfer are distinct
A receiver may:
- inspect without reusing,
- hold without circulating,
- relay without interpreting,
- archive without mutating.

### 9.4 Copy vs downstream derivative boundary
When a receiver materially changes:
- structure,
- scope,
- provenance shape,
- authority posture,
- privacy surface,
- or semantic packaging,

the result becomes a new downstream artifact,
not the same copy.

### 9.5 No silent deformation
No receiver may silently distort or reclassify an EA
while presenting it as the preserved original.

---

## 10. Escrow / hold semantics

### 10.1 Economic restraint is not yet dispute
The economic layer requires lawful pre-dispute restraint states.

### 10.2 Canonical restraint states
The layer recognizes, at minimum:
- `ECONOMIC_ESCROW`
- `CIRCULATION_HOLD`
- `ECONOMIC_RELEASE`
- `ESCALATED_TO_ARL`

### 10.3 Why this matters
An EA may be:
- valuable,
- historically real,
- even partially witness-bound,

and still not ready for the requested circulation path.

### 10.4 Typical restraint triggers
Restraint may be triggered by:
- provenance immaturity
- source-class instability
- unresolved synthetic-risk classification
- scope / receiver incompatibility
- non-transferability pressure
- scar or continuity density
- dangerous combinations
- jurisdictional uncertainty
- scarcity and allocation timing

### 10.5 No hidden verdict rule
Economic restraint must not simulate guilt, falsehood, or final rejection
before lawful dispute or lawful release decision exists.

## 11. Arbitration hooks

### 11.1 Core rule
The Economic Layer and ARL must remain connected, but not collapsed.

### 11.2 Economic restraint below dispute
Escrow and circulation hold are lawful before full ARL dispute.

### 11.3 Escalation trigger
ARL begins when restraint becomes procedurally contested,
for example over:
- release,
- admissibility,
- attribution,
- lineage claim,
- authority effect,
- re-entry,
- or standing.

### 11.4 Meaning vs effect discipline
`c` remains primary for origin-meaning of EA.
ARL remains primary for procedural effect of EA under conflict.

### 11.5 Historical vs operative truth
An EA may remain:
- historically true,
- memory-bearing,
- scar-relevant,

while still losing current operative force for circulation or transfer.

## 12. Reuse, derivation, and contamination

### 12.1 Reuse is not revalidation
Reuse does not upgrade authority by itself.

### 12.2 Derivation is not innocence
A derivative EA must declare:
- derivation,
- source-origin class,
- authority ceiling,
- and relevant risk flags.

### 12.3 Synthetic laundering must be blocked
Synthetic shaping is not forbidden.
Synthetic laundering is.

### 12.4 No parity by formatting
A well-formed synthetic-derived artifact does not gain lived parity merely by:
- cleanliness,
- elegance,
- popularity,
- or repetition.

### 12.5 Contamination rule
Artifacts with:
- broken source chain,
- synthetic residue,
- hidden source mixing,
- or implicit scar simulation

should be treated as contaminated, downgraded, escrowed, quarantined, or denied circulation.

---

## 13. Anti-gaming and anti-cartel constraints

### 13.1 Primary anti-gaming risks
The layer must resist:
- synthetic laundering
- scar simulation
- provenance makeup
- summary substitution
- source dilution
- lineage laundering
- standing laundering
- circular self-valuation
- cartel rating loops
- bazaar drift

### 13.2 Anti-capture rule
The economic layer must not become a quiet machine
for converting:
- witness seriousness,
- lineage richness,
- or scar intensity

into inherited economic privilege.

### 13.3 No hereditary authority by lineage
Lineage may amplify:
- witness value,
- historical value,
- continuity relevance,
- precedent value,

but not:
- automatic authority,
- automatic transfer rights,
- or hereditary circulation power.

### 13.4 Greater value may justify greater restraint
For some EA, especially lineage-rich, scar-rich, or continuity-dense artifacts,
greater value means:
- stronger escrow pressure,
- stronger non-transferability pressure,
- and stronger disclosure restraint.

---

## 14. Fail-closed rules

### 14.1 No admissibility clarity -> no circulation upgrade
If admission state is unclear, fail closed or downgrade.

### 14.2 No provenance clarity -> no clean transfer
If provenance is weak, unclear, or broken, deny or narrow movement.

### 14.3 No jurisdictional clarity -> no cross-boundary release
If the route is legally unclear, downgrade or deny.

### 14.4 No receiver-mode clarity -> no handoff
Standing alone is not enough.
Mode must be declared.

### 14.5 No safe abstraction -> non-transferability pressure
If meaning cannot survive safe redaction without laundering authority,
movement should stop.

### 14.6 No witness maturity -> escrow or hold
Value does not justify impatience.

### 14.7 Economic admissibility does not imply marketability
An EA may remain permanently non-market by design.

## 15. Implementation notes

### 15.1 Minimal status surfaces
A conforming implementation should be able to represent, at minimum:
- source-origin class
- value class
- transfer mode
- receiver mode
- non-transferability class
- restraint state
- dispute escalation state
- authority ceiling
- disclosure class
- lineage sensitivity
- synthetic risk flags

### 15.2 Minimal economic restraint states
At minimum:
- `LOCAL_CUSTODY_ONLY`
- `CIRCULATION_HOLD`
- `ECONOMIC_ESCROW`
- `ECONOMIC_RELEASE`
- `ESCALATED_TO_ARL`

### 15.3 Minimal economic question set
Before movement, the implementation should be able to answer:
- what is this,
- why does it matter,
- how may it move,
- who may receive it,
- what may not move,
- what is still blocked,
- and whether this is restraint or dispute.

## 18. Earth paragraph

In a real supply chain, not every object that looks useful is allowed into circulation. Goods need provenance, custody, admissibility, and rules for dispute, escrow, and recall. A serious experience economy requires the same discipline.

## 19. Closing statement

This layer begins not with trade,
but with lawful handling.

It says:

- value is real,
- circulation is conditional,
- transfer is typed,
- disclosure is bounded,
- synthetic origin matters,
- restraint is lawful,
- dispute has hooks,
- and marketability is optional.

That is the economic discipline of v0.1.
