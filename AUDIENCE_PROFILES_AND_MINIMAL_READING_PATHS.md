# Audience Profiles and Minimal Reading Paths

## Purpose

This file fixes compact canonical reading bundles for the main reader and reviewer types that recur across the corpus.
It helps readers reach bounded first serious orientation without pretending that one path fits everyone or that partial reading authorizes unlimited inference.
Audience profile fixes who should read what first; export profile fixes what may be handed off externally for a bounded task.

## 1. Why this file exists

A mature corpus should not force every reader through the same path.
Different readers need different minimal bundles, but those bundles must remain canonically bounded.

## 2. Reading rule

Minimal reading path is sufficient for first serious orientation, not for unlimited inference.
Do not overread beyond the declared scope of the bundle.

## 4. Anti-overread notes

- narrative-first reading is not equivalent to normative review
- runtime-first reading is not equivalent to doctrinal review
- citation/control-layer reading is not equivalent to full theoretical understanding
- minimal path gives bounded orientation, not unlimited authority to summarize the whole corpus
- audience profile is not the same thing as export / handoff profile
