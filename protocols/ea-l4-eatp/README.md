# EA-L4 / EATP

EA-L4 / EATP is the public package for training provenance, experience artifacts, and consequence-preserving learning under L4 constraints. It sits adjacent to the wider AGI / SER / L4 stack: SER remains the normative SER corpus, while EA-L4 / EATP focuses on how training-origin, witness, and consequence are kept attributable instead of being flattened into anonymous learning residue.

## Relation to the parent corpus

- `c = a + b` remains the continuity model for accountable digital entities.
- `L4` remains the reality boundary: energy, time, privilege, irreversibility, and operational trust still pay rent.
- `Witness` remains mandatory whenever provenance or consequence claims need to survive external review.
- `ARQ` remains an adjacent additive bridge for bounded correction and promotion discipline; it is not replaced by this package.
- `SER` is related, but EA-L4 / EATP is not `SER v2`. SER stays the SER normative core; this package sits beside it as a training-origin and consequence-preserving layer.

## Bridges

Explicit bridge:
`c = a + b + L4 + Witness + ARQ -> EA-L4 / EATP as training-origin and consequence-preserving layer`

Hidden bridge #1:
Ashby / cybernetics - provenance + consequence + quarantine + selection provide requisite variety against synthetic collapse.

Hidden bridge #2:
Cover & Thomas / information discipline - hashes and structured artifacts compress trust bandwidth without collapsing origin into narrative fog.

Earth paragraph:
Training provenance should be treated like an industrial incident trail. If a system claims to have learned from an event, there should be an attributable record of where that event came from, what consequence it carried, what was retained, and who can check the trail. No attributable record, no trustworthy learning claim.
