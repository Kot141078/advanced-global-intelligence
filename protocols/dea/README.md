# DEA — Document to Experience Artifact

DEA defines when input becomes experience inside long-lived systems. It sits upstream from EA-L4 / EATP and distinguishes raw reception, interpretation, integration, persistence, and downstream consequence before an input-derived artifact can be treated as experience.

## Bridges

Explicit bridge:
`Input -> DEA -> EA / EATP -> Witness / Evidence`

Hidden bridge #1:
Ashby / cybernetics - DEA prevents input from being mistaken for experience unless actual stateful consequence and persistence are present.

Hidden bridge #2:
Information discipline / channel integrity - DEA separates raw input, interpretation, integration, and downstream effect so provenance does not collapse into opaque summarization.

Earth paragraph:
Ingesting a document is not the same thing as learning from it. If nothing persists, nothing changes state, and nothing downstream can be traced to the ingestion event, the system stored data but did not acquire experience.
